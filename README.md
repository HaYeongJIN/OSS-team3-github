## [Git 기초 개념 및 명령어](git-concept-command.md)

### Git 기초 개념

Git은 프로젝트의 상태 변화를 '스냅샷' 형태로 기록하는 **분산 버전 관리 시스템**입니다. 단순히 파일을 저장하는 것을 넘어, 시간의 흐름에 따라 누가, 언제, 무엇을 바꿨는지 추적할 수 있게 해줍니다.

### 핵심 명령어 요약

- `git init`: 현재 폴더를 Git이 관리하는 저장소로 초기화합니다.
- `git status`: 파일의 수정 여부와 '준비 상태'를 실시간으로 확인합니다.
- `git add`: 변경된 파일 중 다음 저장 기록(Commit)에 포함될 것들을 골라냅니다.
- `git commit`: 선택된 변경 사항을 하나의 기록으로 영구히 저장합니다.
- `git log`: 지금까지 쌓인 프로젝트의 모든 변경 이력을 확인합니다.

## [Github의 특징 및 브랜치(Branch) 전략](github-branch.md)

### Github의 특징

Github는 Git으로 관리되는 프로젝트를 온라인상에 올릴 수 있는 **원격 저장소 서비스**입니다. 웹 인터페이스를 통해 팀원들과 소통하며 코드를 개선할 수 있는 강력한 기능을 제공합니다.

### 브랜치 전략

브랜치(Branch)는 나뭇가지가 뻗어 나가듯, 하나의 프로젝트 안에서 독립적인 작업 공간을 만드는 기능입니다.

- `main`: 언제든 배포 가능한 가장 안정적인 최상위 상태를 유지합니다.
- `feature branch`: 새로운 기능을 만들거나 고칠 때 생성하며, 작업이 끝나면 다시 `main`에 합칩니다.
- **장점**: 실험적인 코드를 작성하다가 실수를 해도 `main` 브랜치에는 전혀 영향을 주지 않아 안전합니다.

## Github 협업 워크플로우

여러사람이 한 프로젝트를 만들 때 발생하는 충돌을 방지하기 위한 표준적인 흐름을 익히는 것이 중요합니다.

### 실전 협업 단계

1. Clone: 원격에 있는 프로젝트를 내 컴퓨터로 그대로 복제해옵니다.
2. Branch 생성: 나만의 작업 공간을 만듭니다.
3. Work & Commit: 열심히 작업하고 기록을 남깁니다.
4. Push: 내 기록을 다시 Github(원격 저장소)에 올립니다.
5. Pull Request(PR): "내가 만든 코드가 괜찮으니 합쳐줘!"라고 팀원들에게 요청합니다.
6. Review & Merge: 코드 검토 후 최종적으로 프로젝트에 반영합니다.

## 프로젝트 및 이슈 관리

Github는 단순한 코드 저장소를 넘어 강력한 **프로젝트 관리 도구**이기도 합니다.

- Issue: 버그 리포트, 기능 제안, 회의 안건 등을 '티켓'형태로 만들어 관리합니다. 모든 논의 과정이 기록되어 나중에 검색하기 편리합니다.
- Milestone: 여러 이슈를 그룹화하여 프로젝트의 전체적인 진척도(%)를 시각적으로 확인합니다.
- 키워드 자동화: PR 메시지에 `Fixes #12`와 같이 작성하면, 코드가 합쳐지는 순간 12번 이슈가 자동으로 종료되어 관리가 매우 편리합니다.
